# FailBrief

**AI-Powered CI/CD Failure Intelligence for GitHub Actions**

Turn noisy CI logs into actionable insights. FailBrief automatically analyzes failed workflows and delivers smart summaries directly on your pull requests.

---

## 🚀 What is FailBrief?

FailBrief is a GitHub App that transforms how you debug CI/CD failures. Instead of digging through thousands of lines of logs, get instant AI-powered analysis with:

- **Error Classification** - Automatically categorize failures by type (build, test, dependency, lint, etc.)
- **Severity Assessment** - Understand which failures need immediate attention
- **Root Cause Analysis** - AI-generated summaries explaining what went wrong
- **Fix Suggestions** - Actionable recommendations to resolve issues faster
- **Historical Analytics (coming soon)** - Track failure patterns, flaky tests, and CI health over time

## 📦 How It Works

1. **Install** the FailBrief GitHub App on your repositories
2. **Automatic** monitoring of all workflow runs
3. **AI Analysis** kicks in when a workflow fails
4. **Smart Summary** posted as a comment on the relevant PR

No configuration required. Works out of the box.

## 🔒 Security & Privacy

FailBrief takes security seriously:

- **Read-Only Access** - We only read workflow logs, never modify your code
- **No Source Code Storage** - Your code stays in your repository
- **Encrypted Data** - All data encrypted in transit and at rest
- **Minimal Permissions** - Only requests what's necessary
